# Communication Logs

One file per Krishna cycle (or per standalone requirement worked outside a full cycle), named `YYYY-MM-DD_HH-MM-SS.md` for when the cycle started. Every agent that participates in that cycle appends its before/during/after reports here, in the order the work actually happened — agents work one task at a time, not in parallel, so this file reads as a real chronological transcript of how the team moved a batch of requirements from idea to shipped.

Full convention: see "Communication logging" in `docs/agent-collaboration-protocol.md`.

Don't edit or delete another agent's entries. Append only.
