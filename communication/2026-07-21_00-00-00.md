**ISSUE 1 (Critical) — DischargePanel "Save & Complete" always returns HTTP 400**
- DischargePanel.tsx handleSave() calls createDischargeSummary(appointmentId, payload) without first marking the appointment as Completed.
- Backend create_discharge_summary validates appointment.status != "Completed" and raises 400 immediately.
- The appointment is Scheduled when the panel opens. Every Save & Complete submission fails.
- Fix required in backend: change discharge POST to accept Scheduled appointments and mark Completed within the same transaction per OI-8 atomicity. The current "Skip — just mark Complete" path correctly calls updateAppointmentStatus first; the "Save & Complete" path must do this atomically inside the endpoint.

**ISSUE 2 (Critical) — Waitlist confirmation always returns HTTP 409**
- confirm_waitlist_slot uses get_available_slots() to check if the held slot is still free.
- get_available_slots() step 6 excludes slots held by Notified entries — including the confirming patient's own entry.
- So entry.held_slot_time not in available is always True → 409 on every confirm attempt. No patient can ever confirm a waitlist slot.
- Fix required: replace the get_available_slots() call with a direct Appointment query checking whether any actual appointment row (status in Scheduled/Completed) occupies the same doctor/date/time as the held slot.
