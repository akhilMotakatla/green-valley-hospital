**After work completes**

**ISSUE 1 (Critical) ‚Äî DischargePanel "Save & Complete" always returns HTTP 400**
- `DischargePanel.tsx` `handleSave()` calls `createDischargeSummary(appointmentId, payload)` without first marking the appointment as Completed.
- Backend `create_discharge_summary` validates `appointment.status != "Completed"` ‚Üí 400 immediately.
- The appointment is Scheduled when the panel opens ‚Üí every Save & Complete submission fails.
- Fix required in backend: change discharge POST to accept Scheduled appointments and mark Completed within the same transaction (per OI-8 atomicity).

**ISSUE 2 (Critical) ‚Äî Waitlist confirmation always returns HTTP 409**
- `confirm_waitlist_slot` uses `get_available_slots()` to check if the held slot is free.
- `get_available_slots()` step 6 excludes slots held by Notified entries ‚Äî including the confirming patient's own entry.
- So `entry.held_slot_time not in available` is always True ‚Üí 409 on every confirm attempt.
- Fix required: replace `get_available_slots()` with a direct Appointment query checking whether any actual appointment row occupies the slot.
